# Step 09. "어떻게 하면 ~" 질문 만들기

> 목적: 서비스 방향을 맞춘 뒤, **더 구체적인 생각**으로 수렴한다.

> **Guide:**  
> 1. Agent가 `docs/04`, `docs/06`, `docs/08`을 읽고 **"어떻게 하면 ~ 할 수 있을까?"** 형태의 질문을 만든다. 직접 질문을 추가해도 좋다.
> 2. 팀이 질문을 보며 답을 이야기한다.
> 3. 각 질문 아래 **답**을 적는다. (타이머 ON)  
> - 질문은 **많을수록** 좋고, 하고 싶은 말이 많은 질문이 좋다.  
> - **실제로 어떻게 할지** 떠올릴 수 있는 질문을 한다. (예: 사용자를 어떻게 모을까? → 너무 넓음)  
> - 답은 **질문과 같은 말 반복·뭉뚱그린 표현 금지**. 구체적인 문장으로 쓴다.  
>   - (X) 어떻게 하면 사용자가 이 기능을 더 잘 쓸까? → 사용자들이 더 쉽게 쓰게 만들면 될 것 같아요  
> - 참고한 서비스·자료가 있으면 답에 적는다.

**프롬프트 예시**

```
docs/04_service_goal_target_value.md, docs/06_pain_wow_points.md, docs/08_word_cloud.md를 읽고
docs/09_how_might_we_questions.md의 "Agent가 만든 질문" 섹션만 채워줘.

질문 규칙:
- "어떻게 하면 ~ 할 수 있을까?" 형태
- docs/04 목적·핵심 가치, docs/06 불편한 점·좋았던 점, docs/08 키워드를 골고루 반영
- 실제 방법·아이디어가 떠오를 수 있는 질문 (너무 넓은 "사람 모으기"류 X)
- 최소 10개 이상, 서로 겹치지 않게
- 각 질문 아래 답 칸은 비워둬 (팀이 채움)
- "팀이 추가한 질문" 섹션은 건드리지 마
- 구조와 상단 Guide는 유지
```
