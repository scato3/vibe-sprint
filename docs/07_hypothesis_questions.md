# Step 07. 의심하고 질문 만들기

> 목적: `docs/06`의 Wow가 성립하려면, 우리가 **믿고 있는 전제**가 맞는지 짚어본다.

> **Guide:** Wow 하나마다 아래 3단계를 순서대로 적는다.  
> ① 고객 행동 예측 → ② 그 뒤에 숨은 전제 → ③ 전제가 맞는지 의심하는 질문.  
> 답을 확정하지 말고, 스프린트 중에 검증할 질문을 만든다.

**프롬프트 예시**

```
docs/06_pain_wow_points.md의 Wow를 바탕으로 docs/07_hypothesis_questions.md를 채워줘.
Wow마다 ①기대 행동 ②숨은 전제 ③검증 질문 세트를 작성해줘.
구조와 상단 Guide는 유지하고, 본문만 채워줘.
```
